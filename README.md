## About the Project

**GitMaster Pro** is a feature-rich, browser-based Git tutorial designed for learners at every level. Whether you're writing your first `git init` or mastering interactive rebases, this platform provides hands-on, risk-free practice with instant visual feedback.

Built as a single-page application with **zero backend dependencies**, it runs entirely in your browser — no installation, no sign-up, no risk to real repositories.

### Why GitMaster Pro?

| | Feature | Description |
|---|---|---|
| **Interactive Terminal** | Practice 200+ Git commands in a safe, simulated environment |
| **Visual Git Graphs** | Watch commits, branches, and merges animate in real-time |
| **Multi-OS Support** | Platform-specific instructions for Windows, macOS, and Linux |
| **Bilingual** | Full English and Arabic support with proper RTL layout |
| **Progress Tracking** | Track your learning journey with achievements and completion stats |
| **Offline Ready** | Works without internet after initial load |

## Interactive Terminal

A fully simulated Git terminal running in your browser — no real repositories affected.

**Supported operations include:**
```
git init, add, commit, status, log, diff, branch, switch, checkout,
merge, rebase, stash, cherry-pick, tag, remote, push, pull, fetch,
clone, reset, revert, restore, bisect, reflog, clean, rm, mv ...
```

**25+ Guided Challenges:**

| | Challenge | What You'll Practice |
|---|-----------|---------------------|
| First Commit | `init` &#8594; `add` &#8594; `commit` |
| Branch & Merge | Create branches and merge them |
| Stash Flow | Save and restore work-in-progress |
| Cherry Pick | Apply specific commits across branches |
| Rebase Flow | Rewrite commit history linearly |
| Interactive Rebase | Squash and reorder commits |
| Merge Conflicts | Resolve conflicting changes |
| Git Bisect | Find bugs with binary search |
| *...and 17 more* | |

## Git Visualizer

Watch Git operations come alive with animated, canvas-based commit graphs.

**12 Scenario Presets:**

| Scenario | Description |
|----------|-------------|
| Basic Commits | Linear commit history on a single branch |
| Feature Branch | Branch off, work, merge back |
| Fast-Forward | Pointer-only merge (no merge commit) |
| Rebase | Replay commits onto a new base |
| Cherry-Pick | Apply individual commits |
| Merge Conflict | Three-way merge with conflicts |
| Revert Commit | Safely undo a published commit |
| Reset (Soft/Hard) | Move HEAD and optionally discard changes |
| Stash & Apply | Temporarily shelve changes |
| Tagging Releases | Mark release points |
| Detached HEAD | Explore commits without a branch |
| Hotfix Workflow | Emergency production fix |

**Workflow Strategies Covered:**
- **GitFlow** — Feature/develop/release/hotfix branching model
- **GitHub Flow** — Simplified continuous deployment
- **Trunk-Based Development** — Short-lived feature branches

## Getting Started

### Option 1: Use Online (Recommended)

Visit **[marafah.github.io/git](https://marafah.github.io/git/)** — nothing to install.

### Option 2: Run Locally

```bash
# Clone the repository
git clone https://github.com/marafah/git.git

# Navigate to the project
cd git

# Open in your browser
# Simply open index.html — no build step needed
start index.html        # Windows
open index.html         # macOS
xdg-open index.html     # Linux
```
